About Search

The system provides a search component for differents places in the site. This search component is a powerful Google-like searcher that it would normally find what the user is looking for. Please observe the following rules for its use:

  • All you need to do is to type the search criteria in the text box of the search.
  • To apply a more refined search feel free to use the advanced search form (if available)
  • Or type the criteria for each of the columns

Depending on the type of column, the specific search has different rules:

  • Numeric Search: you can use the greater than, lower than and between operators. The equal operator is implied when no operator is used. Here are some examples:
    • @id 100
    • @height > 25
    • @age < 10
    • @number 30:40

  • Date Search: it should be used just like the Numeric Search except that dates should be represented in the international standard date notation which is:

    YYYY-MM-DD

    where YYYY is the year in the usual Gregorian calendar, MM is the month of the year between 01 (January) and 12 (December), and DD is the day of the month between 01 and 31.
    For example, the fourth day of February in the year 1995 is written in the standard notation as

    1995-02-04

    Additionally, you can use functions ($year, $month and $day) to be applied to the date column. For example, to search for the people for which their birthday is on May, 5th: 
    • $day(@birth) 5 and $month(@birth) 5
    Of course, you can always use the comparisons with these functions.

  • String Search: There are two types of strings, the indexed strings and not indexed strings. The first ones mean that the system knows even how to understand these strings; for example: 
    • Mark Twain
    Should find all ocurrences of 'Mark Twain' as well as 'Twain, Mark'.
    NOT indexed strings can only use a wildcard '*' at the beginning or at the end of a string: 
    • @name Mar*

  • Boolean Search: These are the most simple type of columns. The only values allowed here are: true (or 1) and false (or 0)
